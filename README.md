# Пользовательский алгоритм управления перегрузкой TCP (Cubic Spline CC)

## Обзор

Этот проект представляет алгоритм управления перегрузкой (Congestion Control, CC) для протокола TCP, основанный на модели кубического сплайна. Алгоритм динамически регулирует окно перегрузки (cwnd) с учетом времени кругового обхода (RTT), пропускной способности и истории cwnd, обеспечивая баланс между отзывчивостью и стабильностью. Реализация выполнена на языке C с использованием арифметики с фиксированной точкой для высокой производительности, что делает ее подходящей для встраиваемых систем и приложений с высокими требованиями.

Алгоритм моделирует cwnd через коэффициенты \(a\), \(b\), \(c\), \(d\), формируя кубический сплайн, который адаптируется к сетевым условиям. Ведется разработка для внедрения пороговых значений и зависимости от подтверждений (ACK) для повышения устойчивости в сложных сетях.

## Основные особенности

- **Кубический сплайн**: Плавное и адаптивное управление cwnd.
- **Адаптация к RTT**: Учет минимального и текущего RTT для реакции на изменения задержки.
- **Чувствительность к пропускной способности**: Точная настройка cwnd на основе измерений полосы пропускания.
- **Фиксированная арифметика**: Использование 64- и 32-битных целых чисел с \(FIXED_SHIFT = 10\) для оптимизации.
- **Кэширование**: Снижение вычислительных затрат за счет кэширования соотношений и пропускной способности.
- **Текущая разработка**: Интеграция порогов (\(THRESHOLD_PERCENT\), \(THRESHOLD_PERCENT_HIGH\), \(THRESHOLD_PERCENT_LOW\)) и зависимости от ACK.

## Формулы

Алгоритм использует следующие выражения для вычисления коэффициентов и окна перегрузки:

- **Коэффициент `d`**:  
  Определяет чувствительность к изменениям RTT с учетом кубического отношения.  
  ```
  d = 1 + (loc_rtt + loc_rtt/2)/loc_rtt, где loc_rtt ~ (RTT_curr/RTT_min)^3
  ```
  `loc_rtt` зависит от куба отношения текущего RTT к минимальному.

- **Коэффициент `c`**:  
  Балансирует между текущим и предыдущим окнами перегрузки.  
  ```
  c = min(endl_cof_cwnd, CWND_curr), где endl_cof_cwnd ~ |CWND_last - CWND_curr|
  ```
  `endl_cof_cwnd` учитывает абсолютную разницу между текущим и предыдущим cwnd.

- **Коэффициент `b`**:  
  Отражает пропускную способность, нормализованную относительно `c + d_initial`.  
  ```
  b = T / (T / (c + d_initial))
  ```

- **Коэффициент `a`**:  
  Интегрирует остальные коэффициенты для плавной корректировки.  
  ```
  a = (b + c - d_initial) / 4
  ```

- **Промежуточное окно перегрузки**:  
  Комбинирует коэффициенты для предварительной оценки cwnd.  
  ```
  CWND_inter = a + b + c - d_initial
  ```

- **Следующее окно перегрузки**:  
  Итоговое окно с учетом текущего cwnd и ограничений.  
  ```
  CWND_next = CWND_inter + (2 * CWND_curr) / d_initial (с ограничениями по RTT и CWND_last_max)
  ```
  
## Структура алгоритма


Алгоритм реализован в структуре `SplineCC` (`sCC`), содержащей:
- **RTT**: `min_rtt`, `curr_rtt`.
- **Окно перегрузки**: `curr_cwnd`, `last_cwnd`, `last_max_cwnd`.
- **Пропускная способность**: `throughput`, `throughput_t`.
- **Коэффициенты**: `a`, `b`, `c`, `d`, `d_initial`.
- **Кэширование**: `cached_ratio`, `cached_throughput`, `last_min_rtt`, `last_c_d_initial`.

## Основные функции

Алгоритм состоит из следующих ключевых функций для вычисления коэффициентов и управления окном перегрузки:

- `find_cof_rtt`:  
  Вычисляет коэффициент `d` на основе отношения `RTT_curr/RTT_min` с кубическим масштабированием.  
  ```
  d ~ (RTT_curr/RTT_min)^3
  ```

- `find_cof_cwnd`:  
  Определяет коэффициент `c` с учетом тенденций окна перегрузки и предотвращения резкого роста.  
  ```
  c = min(endl_cof_cwnd, CWND_curr)
  ```

- `find_cof_bw`:  
  Рассчитывает коэффициент `b` на основе нормализованной пропускной способности.  
  ```
  b = T / (T / (c + d_initial))
  ```

- `find_cof_a`:  
  Интегрирует коэффициент `a` из `b`, `c` и `d_initial`.  
  ```
  a = (b + c - d_initial) / 4
  ```.

## Ключевые параметры
- \(FIXED_SHIFT = 10\): Точность фиксированной точки (1024 единицы).
- \(MSS = 1460\): Максимальный размер сегмента.
- \(MAX_RTT = 1,000,000 \mu s\): Максимальный RTT.
- \(MIN_RTT = 10 \mu s\): Минимальный RTT.
- \(THRESHOLD_PERCENT = 120\%\): Базовый порог (в разработке).
- \(THRESHOLD_PERCENT_HIGH = 60\%\): Консервативный порог.
- \(THRESHOLD_PERCENT_LOW = 110\%\): Агрессивный порог.
- \(MAX_RATIO = 2^{21}\): Ограничение переполнения.

## Рабочий процесс
1. **Измерение RTT**: Отслеживание `min_rtt` и `curr_rtt`.
2. **Вычисление коэффициентов**:
   - \(d\): Кубическое масштабирование RTT.
   - \(c\): Тенденции cwnd.
   - \(b\): Нормализованная пропускная способность.
   - \(a\): Интеграция коэффициентов.
3. ## Обновление окна перегрузки

Процесс обновления окна перегрузки включает два этапа:

- **Промежуточное окно**:  
  ```
  CWND_inter = a + b + c - d_initial
  ```
  Формирует предварительную оценку на основе коэффициентов.

- **Следующее окно**:  
  ```
  CWND_next = CWND_inter + (2 * CWND_curr) / d_initial
  ```
  Итоговое окно с учетом текущего cwnd и ограничений по RTT и `CWND_last_max`.
4. **Кэширование**: Оптимизация вычислений.

## Текущая разработка
- **Пороговые значения**: Интеграция \(THRESHOLD_PERCENT\) для ограничения роста cwnd.
- **Зависимость от ACK**: Учет подтверждений для динамической корректировки.
- **Тестирование**: Оценка в различных сетевых условиях.

## Рекомендации по использованию
- **Инициализация**: Корректно задайте `min_rtt`, `curr_rtt`, `curr_cwnd`.
- **Проверка данных**: Алгоритм возвращает безопасные значения при некорректных входных данных.
- **Производительность**: Оптимизирован для систем с ограниченными ресурсами.
- **Масштабируемость**: Поддерживает RTT от 10 мкс до 1 с.

## Пример использования
```c
#include <stdio.h>
#include "spline_cc.h"

int main() {
    sCC state = {0};
    state.min_rtt = 100;      // 100 мкс
    state.curr_rtt = 150;     // 150 мкс
    state.curr_cwnd = 10;     // 10 сегментов
    state.last_cwnd = 8;      // Предыдущее окно
    state.throughput_t = 1000000; // 1 МБ/с

    state.d = find_cof_rtt(state.min_rtt, state.curr_rtt, &state);
    state.c = find_cof_cwnd(state.last_cwnd, state.curr_cwnd, &state);
    state.b = find_cof_bw(state.throughput_t, &state);
    state.a = find_cof_a(&state);
    state.inter_cwnd = resolve_inter_cwnd(&state);
    state.next_cwnd = resolve_next_cwnd(&state);

    printf("Следующее окно: %u сегментов\n", state.next_cwnd);
    return 0;
}
```

## Преимущества
- **Гибкость**: Сплайн-модель адаптируется к различным сетевым условиям.
- **Производительность**: Оптимизирован для встраиваемых систем.
- **Адаптивность**: Чувствителен к изменениям RTT.
- **Потенциал**: Планируемые улучшения повысят конкурентоспособность.

## Недостатки
- **Отсутствие реакции на потери**: Не учитывает потери пакетов.
- **Консервативность**: Ограничивает использование пропускной способности в стабильных условиях.
- **Неполные пороги**: \(THRESHOLD_PERCENT\) пока не интегрирован.
- **Ограниченное тестирование**: Требуется проверка в сложных сетях.

## Ограничения
- Пороговое управление cwnd в разработке.
- Нет прямой корректировки на основе ACK.
- Предполагает стабильные измерения RTT; высокая изменчивость требует доработки.

## Заключение
Cubic Spline CC — инновационный алгоритм, сочетающий гибкость сплайн-модели, учет RTT и пропускной способности, а также оптимизацию для производительности. Он перспективен для встраиваемых систем и высокопроизводительных сетей, но требует доработки для обработки потерь пакетов и интеграции пороговых значений, чтобы конкурировать с CUBIC и BBR в сложных сетях. Тесты подтверждают его адаптивность, но дальнейшее тестирование и улучшения сделают его более универсальным.
